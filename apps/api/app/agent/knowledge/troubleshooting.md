# 常见问题与排查（答疑）

> 用户报「手崩了」「脸糊」「视频漂移」「显存不够」「排队慢」等问题时,助手据此给出有依据的诊断与可执行的修法。
> 参数取值见 parameter-reference.md,模型选择见 model-catalog.md。

## 手崩 / 多指 / 畸形肢体
- 原因:扩散模型对手部/末端结构本就薄弱;尺寸过大或步数过低会加剧。
- 修法:
  1. 负面词补 `bad hands, extra fingers, fused fingers, missing fingers, mutated hands, extra limbs, deformed`。
  2. 适度提高 steps(20→30),cfg 别太高(7 左右),太高反而僵硬出伪影。
  3. 让手不占画面主体(远景/手藏起来/换姿势)能显著降低翻车率。
  4. 已出好图只有手坏:用 img2img 局部低强度重绘(denoise 0.3–0.4),或自定义工作流上 Impact 的局部修复(手部/脸部 Detailer)。
  5. 多出几张(batch_size=4)挑手好的一张,通常比死磕一张高效。

## 脸糊 / 脸崩 / 五官乱
- 原因:脸在画面里占比太小(远景),或尺寸超出模型训练分辨率,细节不够。
- 修法:
  1. 写实脸优先用 majicMIX realistic;二次元脸用 Illustrious 系,选对底模一半问题就没了。
  2. 负面词补 `bad face, deformed face, asymmetric eyes`;提示词加 `detailed face, sharp focus`。
  3. 脸太小 → 拉近构图(portrait、close-up)或先小图出好再放大精修。
  4. 自定义工作流用 FaceDetailer(Impact)对脸部局部重绘,是修脸最有效的手段。

## 出图偏题 / 不听提示词
- cfg 太低(<4)→ 提到 6–8 让它更服从;但太高(>12)会过曝僵硬。
- 提示词太长太散 → 精简,把核心主体提到最前。
- 关键元素被忽略 → 调整词序(靠前权重高),或拆成更明确的短语。
- 用了不对口的模型 → 二次元词喂写实模型(或反之)会拧巴,先选对底模。

## 出图过曝 / 颜色发灰 / 死黑
- cfg 过高 → 过曝/僵硬;适当降到 6–8。
- 负面词补 `overexposed, oversaturated`(过曝)或 `low contrast, grayscale`(发灰)。
- 换 sampler/scheduler(如 dpmpp_2m + karras)有时能改善整体观感。

## 视频漂移 / 闪烁 / 运动鬼影
- 原因:Wan 4 步加速档下要求的运动太剧烈/太复杂,或多主体同时大幅运动。
- 修法:
  1. 把运动描述改「连续、温和、单一」:`slow pan, gentle drift, subtle motion`,去掉剧烈动作词。
  2. 首帧(底图)质量决定上限 → 先把底图出好再图生视频。
  3. 缩短时长(更少 length)通常更稳;别一上来就拉满 121 帧。
  4. 尺寸别太大,640×480 是稳妥点。
- 注意:视频的 steps=4 / cfg=1 / scheduler=simple 是加速方案的固定配套,改它们一般会让效果更差,不建议动。

## 视频/3D/音乐「超时」或没出来
- 这些任务本就慢:Wan 视频 1–2 分钟、Hunyuan3D 约 1–3 分钟、长音乐也要时间,先耐心等。
- 平台对话内有等待上限(视频/3D 比图像长很多),偶发超时多为排队或正好赶上节点忙 → 稍后重试。
- 自定义工作流没产物:八成是图里**缺 Save 类节点**(SaveImage/SaveAnimatedWEBP/SaveAudioMP3/SaveGLB),补上即可。

## 显存 / OOM(显存不够)
- 平台是 4× RTX PRO 6000(每卡显存很大)+ 5090,常规出图很难 OOM;真遇到多半是自定义工作流太重。
- 修法:
  1. 降尺寸(图像回 512 系、视频回 640×480)、降 batch_size、降 length。
  2. 视频/3D 用默认尺寸;别在一张图里堆多个大模型(多 checkpoint + 多 ControlNet + 高分辨率)。
  3. 优先 fp8/蒸馏版模型(平台 Wan/Qwen-Image/Flux 多为 fp8),显存更省。

## 排队慢 / 一直在等
- 平台多机异构调度:提交时只挑「同时具备所需全部模型」且较空闲的 worker。
- 视频/3D 这类大模型只装在部分节点,可用节点少时会排队,属正常。
- 不必反复重交(重复占队列);耐心等或错峰再试更好。

## 「找不到模型 / worker 不可用」类报错
- 自定义工作流里写了平台没装的模型文件名 → 用 list_models 查真实 checkpoint 名,或 search_knowledge 查配方里的准确文件名,别凭记忆编。
- 图生图/图生视频/图生3D 报「worker 不可用」→ 图片上传所在的那台 worker 当前不可用,重新上传一次图片即可(这类任务必须在图片所在的 worker 上跑)。

## 复现某张满意的图
- 每次生成都会回传实际 seed。固定该 seed + 完全相同的提示词/参数/模型 → 复现同一张。
- 想在它基础上微调:固定 seed,只小改提示词或 cfg,变化会很克制可控。
- 注意:带 _ancestral / _sde 的采样器随机性更强,复现性不如 euler/dpmpp_2m。

## 中文出不来 / 画面里文字乱
- SD1.5 等模型几乎画不出准确中文 → 要画面里有中文字,改用 **Qwen-Image**,并直接在提示词里写出要出现的中文内容。
- 即便如此,长句中文仍可能出错,尽量短词/短句,必要时后期叠字更稳。
