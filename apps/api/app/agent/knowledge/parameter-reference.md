# 参数详解（与平台真实默认/约束一致）

> 逐个讲清生成参数:作用、好的取值范围、调高/调低的影响、常见搭配。
> 文中的默认值与上下限严格对应 ToIV 后端实际代码(routes/generate.py、audio/video/threed.py、各 workflow 构造器),
> 助手据此给建议时不要超出这些约束,否则提交会被拒。

## 通用图像参数（txt2img / img2img,SD1.5 系)
适用 generate_image / edit_image,以及自定义 txt2img/img2img 工作流。

### steps(采样步数)
- 作用:去噪迭代次数,越多细节越充分。
- 默认 20;允许 1–150。
- 调高(30–40):细节更稳、收敛更好,但更慢,过高(>50)边际收益很小。
- 调低(10–15):快,但可能糊/不收敛。
- 搭配:SD1.5 日常 20 足够,要质量上 30;Z-Image Turbo / Flux 这类蒸馏模型只需 4–8 步,别套 20+。

### cfg(提示词引导强度)
- 作用:画面对提示词的服从程度。
- 默认 7.0;允许 0–30。
- 调高(9–12):更贴提示词但易过曝、僵硬、出伪影。
- 调低(3–5):更自然、发挥更大,但可能偏题。
- 搭配:SD1.5 稳妥 7;写实人像可 5–7;新架构(Flux/Z-Image)用很低 cfg(常 1–4),Wan 视频 cfg=1。

### sampler(采样器,sampler_name)
- 作用:去噪算法。
- 默认 euler;常见还有 euler_ancestral、dpmpp_2m、dpmpp_2m_sde、dpmpp_3m_sde、uni_pc、ddim 等。
- 影响:euler 稳;dpmpp_2m 在中等步数下质量好;带 _ancestral/_sde 的更有变化但同 seed 不易复现。
- 搭配:SD1.5 默认 euler+normal 最稳;要更细腻可试 dpmpp_2m + karras。

### scheduler(噪声调度)
- 作用:每步噪声衰减曲线。
- 默认 normal;常见还有 karras、simple、sgm_uniform、exponential、beta。
- 影响:karras 低步数下细节更好,常配 dpmpp 系;simple 是 Wan/ACE-Step 的默认。
- 搭配:SD1.5 图像默认 normal;Wan 视频 / ACE-Step 音频默认 simple(平台已设好,别乱改)。

### seed(随机种子)
- 作用:决定初始噪声;同参数+同 seed → 可复现同一张图。
- 默认随机(范围 0 ~ 2^63-1,即 9223372036854775807)。
- 用法:满意某张图想微调,固定它的 seed 再改提示词/参数;要多样性就让它随机。
- 提示:平台每次生成都会回传实际 seed,记下来即可复现。

### denoise(重绘强度,仅 img2img / edit_image)
- 作用:对输入图保留多少。值越大改动越大、越不像原图。
- 默认 0.6;允许 0.1–1.0(txt2img 内部恒为 1.0,不需设)。
- 取值指南:0.3–0.4 轻微改(调色/微调) | 0.5–0.6 中等改(换风格保构图) | 0.7–0.85 大改 | 0.9+ 几乎重画。
- 搭配:想「换风格但保住构图/人物姿态」→ 0.4–0.55;想「只借个轮廓重画」→ 0.75+。

### width / height(画幅)
- 作用:输出分辨率。
- 默认 512×512;允许 64–2048,后端会自动吸附到 8 的倍数(SD 潜空间要求)。
- SD1.5 推荐:总像素接近 512×512,用 512 的倍数,如 512×512 / 512×768(竖) / 768×512(横) / 768×768。
- 影响:远超训练分辨率(如 SD1.5 直接上 1024+)易出现重复肢体/构图崩;要高清优先小图出好再放大(hires/img2img)。
- 助手画幅档(generate_image 的 aspect):1:1=512×512、2:3=512×768(竖)、3:2=768×512(横)、hd=768×768。

### batch_size(批量)
- 作用:一次出几张(同提示词、不同 seed)。
- 默认 1;允许 1–8。
- 用法:挑构图/选脸时一次出 4 张对比;越大越占显存越慢。

## 视频参数（Wan 2.2 I2V）
适用 generate_video 及 /generate/video。平台视频是「先文生底图 → Wan 驱动运动」。

### length(帧数)
- 作用:视频总帧数,决定时长(时长 ≈ length / fps)。
- 默认 49 帧;允许 9–121。约束:Wan 需 4n+1 帧(如 49、81、121);助手生成时会自动吸附。
- 影响:越多越长越慢;16fps 下 49 帧≈3 秒,81 帧≈5 秒,121 帧≈7.5 秒。
- 对话内 generate_video 的 seconds 入参范围 1–7.5 秒,内部换算成合法 length。

### fps(帧率)
- 作用:播放速度。默认 16;允许 4–30。
- 影响:同 length 下 fps 越高越「快而短」,越低越「慢而长」;Wan 训练偏 16,默认即可。

### width / height(视频画幅)
- 默认 640×480;允许 128–1280。
- 提示:Wan I2V 底图与视频同尺寸;加速档下别上太大尺寸,640×480 是稳妥点。

### steps / cfg / scheduler(视频,通常不用动)
- 默认 steps=4(lightx2v 4 步加速 LoRA)、cfg=1.0、sampler=euler、scheduler=simple。
- 这是加速方案的配套设定,平台已固定;改动易破坏加速效果,不建议用户调。

### 视频提示词与运动
- 描述画面+「连续、温和」的运动:slow pan、gentle wind、drifting、flowing water、subtle motion。
- 4 步加速下避免要求剧烈/复杂/多主体运动,简单连续运动最稳。

## 3D 参数（Hunyuan3D 图生3D）
适用 generate_3d 及 /generate/3d。输入一张图,输出 GLB。

### steps
- 作用:3D 扩散采样步数。默认 30;允许 10–100。
- 影响:越多形状越精细但越慢;30 为质量/速度平衡点。

### cfg
- 作用:对输入图条件的服从度。默认 5.0;允许 0–20。
- 影响:过高可能僵硬/伪影,5 左右稳妥。

### octree_resolution(网格分辨率)
- 作用:体素八叉树分辨率,决定 mesh 精细度。默认 256;允许 64–512。
- 影响:调高(384/512)细节更多、面数更大、更慢、文件更大;调低(128)更快更糙。
- 搭配:展示初稿用 256;要更精细的成品再上 384/512。

### 其它(内部固定)
- resolution=3072(潜空间)、num_chunks=8000、threshold=0.6、sampler=euler、scheduler=normal,平台已设好。
- 输入图建议:主体居中、背景干净、单一物体,3D 重建效果最佳。

## 音频参数（ACE-Step 文生音乐）
适用 generate_music 及 /generate/audio。

### tags(风格标签,必填)
- 作用:决定流派/乐器/情绪/节奏,是音乐的「灵魂」。
- 写法:逗号分隔关键词,如 `lofi, chill, piano, 90bpm`、`epic orchestral, cinematic, drums`、`synthwave, retro, 120bpm`。
- 建议含:流派 + 主要乐器 + 情绪 + BPM(节奏),信息越具体越可控。

### lyrics(歌词,可选)
- 作用:留空=纯音乐;填写=带演唱的歌曲。
- 写法:可分段(主歌/副歌),中英文均可;歌词太长会拉长结构。
- 配套:内部 lyrics_strength=1.0 控制对歌词的贴合度(平台固定)。

### seconds(时长)
- 作用:音乐长度。默认 30;允许 5–240 秒。
- 影响:越长越慢;短曲(15–30s)适合片头/BGM 循环,长曲(60s+)适合完整段落。

### steps / cfg(通常不用动)
- 默认 steps=50(允许 10–150)、cfg=5.0(允许 0–20)、sampler=euler、scheduler=simple。
- 步数高些(50)利于音质;cfg 5 平衡贴合与自然。
