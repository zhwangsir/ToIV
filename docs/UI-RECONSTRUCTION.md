# ToIV UI 重构设计报告
> 2026-06-24 · 目标:从"再调一版"升级为方向性重构,把平台从"像 AI 网站"做到"像专业创作工具"。

---

## 0. TL;DR(给赶时间的你)

当前 UI 不美观**不是配色深浅的问题,是四个结构性硬伤**:① 暖金 + 近黑 = AI 产品最泛滥的套路;② **中文大标题其实没字体**——Fraunces 是拉丁衬线,中文回退到系统苹方,所谓"杂志衬线刊头"在中文上根本没生效;③ chrome(大字/水印/渐变)抢了作品的戏,而平台的核心是图/视频;④ 缺真景深 + 重复的"巨标 + 序数水印"空态 gimmick = 模板感。

重构北极星:**作品至上 · 精度大于装饰 · 强调色当墨水不当霓虹 · 给中文真正的字体**。下面给三个方向(**A 暗房工作室 / B 极光玻璃 / C 明亮画廊**),推荐 **A**,并附 A 的完整设计系统 + 分阶段实施计划。**最后一节是需要你拍板的三个选择。**

## 1. 诊断:为什么现在"不美观"(基于实读 `globals.css` v5)

| # | 现状 | 问题 | 严重度 |
|---|---|---|---|
| 1 | `--bg-0:#0c0b0d` 暖近黑 + `--accent:#ecaa56` 琥珀金,到处 `--accent-glow` | **"深色 + 暖色发光"是 AI 站套路**,一眼判定"AI 工具",廉价感来自这里 | 高 |
| 2 | `--font-display: "Fraunces"...` 用在 14+ 处大标题 | **Fraunces 不含中文**。中文标题全部回退到 `PingFang/YaHei` 系统字 → 你看到的"杂志大标"其实是系统苹方放大,生硬、没设计;拉丁眉标是 Fraunces、中文是苹方 → 同屏两套气质打架 | **致命** |
| 3 | 每个视图 `.view-header`(巨标)+ `.editorial-empty`(序数水印) | 同一套"刊头 + 大水印"在 6 个视图重复 → 模板感、gimmick 感,重复即廉价 | 高 |
| 4 | 媒体区(作品库瓦片、创作台 stage)偏小、偏暗、空 | **平台核心是图/视频,UI 却让 chrome 当主角**、把作品压成配角 | 高 |
| 5 | 面板=扁平深色块 + 描边;`--shadow-*` 有但弱 | 缺成体系的景深(层叠/玻璃/光边),"高级感"恰恰来自精确的层次 | 中 |
| 6 | Inter 正文,中文回退苹方;无数据等宽字 | 专业工具该有 mono 呈现 seed/参数/分辨率;现在数字和正文一个字体,缺"工程精度感" | 中 |
| 7 | `--radius/spacing/shadow` 有 token,但组件实际用法不统一 | 圆角/间距/高度不成严格刻度 → 细看处处差半像素,累积成"不精致" | 中 |

**一句话**:不是颜色丑,是 **缺真正的字体系统(尤其中文)+ 作品没当主角 + 强调色滥用发光 + gimmick 重复**。

## 2. 设计北极星(重构准则)

1. **Work is the hero / 作品至上** — chrome 退场,媒体放大、铺满、留白衬托。导航和参数是"乐谱",作品是"演出"。
2. **Precision over decoration / 精度大于装饰** — 严格的 8pt 刻度、统一圆角阶、统一高度阶;高级感来自一致与精确,不来自特效。
3. **One confident accent, used as ink / 一个强调色,当墨水** — 强调色只用在"此刻最重要的一个动作/状态",不做背景辉光、不到处渐变。
4. **Type as craft / 字体即工艺** — 给中文真正的显示字 + 文本字 + 数据等宽字,三者分工。这是单点提升感知质量最大的一步。
5. **Systematic depth / 成体系的景深** — 0–4 级高度,每级 = 背景 + 光边 + 阴影 + (可选)模糊的确定组合。
6. **Restraint / 克制** — 删掉重复 gimmick,空态做"安静且有用"。

## 3. 三个候选方向

> 都适合"专业 AIGC 工具 + NSFW 主旋律",但气质与风险不同。每个都给了真实参照、为什么、风险。

### 方向 A — Studio Noir / 暗房工作室 ★推荐
- **气质**:中性近黑(非暖、微冷)、极克制的单一精确强调、大量媒体、真景深(细光边 + 柔阴影 + 少量玻璃)、grotesk UI 字 + mono 数据字。
- **参照**:Linear、Midjourney web、Runway、Vercel/Geist。
- **为什么**:暗房最适合长时间看图调色 + 衬托作品(尤其 NSFW);中性色 + 克制强调最"高级、不 AI";和你喜欢的 `/engine` 科幻感不冲突——把 `/engine` 当**入口/营销层的高能**,工作区回归**安静的专业**。
- **风险**:低。执行重点在"精度"而非"想象力",可控。

### 方向 B — Aurora Glass / 极光玻璃
- **气质**:深底 + 分层半透明玻璃面板 + 真高斯模糊景深 + 光谱(多色相)强调,克制使用。
- **参照**:visionOS、Arc Browser、Apple 新版 UI。
- **为什么**:延续 `/engine` 的科幻交互感,"充满交互/未来感";现代、有 wow。
- **风险**:中高。玻璃 + 模糊 = 性能成本;把控不好又滑回"AI 发光"。需要强纪律。

### 方向 C — Bright Atelier / 明亮画廊
- **气质**:暖米白画布、ink 黑字、Swiss 栅格、一个强烈强调、作品装裱在优雅画框里。
- **参照**:Krea(亮)、Cosmos、美术馆官网、作品集站。
- **为什么**:最"不像 AI"、最编辑/高级、最少撞脸。
- **风险**:高。亮底 + 露骨/暗调 NSFW 作品观感冲突;暗房工作流(长时间看图)亮色更累眼。建议**最多作为可切换的第二主题**,不作默认。

## 4. 推荐方向 A 的完整设计系统(可直接当重构规范)

### 4.1 调色板(oklch,暗房中性 + 单一精确强调)
> 关键变化:**从"暖近黑"转到"中性微冷近黑"**(去掉 AI 味的暖膜);强调色给两个候选,见 §6 让你选。

```css
:root {
  color-scheme: dark;
  /* 表面:中性近黑,层层抬升靠明度不靠色相 */
  --bg-canvas:  oklch(15% 0.006 265);  /* 页面底,几乎纯中性、极微冷 */
  --bg-raised:  oklch(19% 0.006 265);  /* 卡片/输入 */
  --bg-overlay: oklch(23% 0.007 265);  /* 弹层/悬浮 */
  --bg-sunken:  oklch(12% 0.005 265);  /* 凹陷区(画布、媒体衬底) */

  /* 文字:中性暖白四档,纪律分明 */
  --ink:       oklch(96% 0.004 90);
  --ink-2:     oklch(72% 0.004 90);
  --ink-3:     oklch(54% 0.004 90);
  --ink-faint: oklch(42% 0.004 90);

  /* 描边:做层次的主力,中性 */
  --line:        oklch(100% 0 0 / 0.08);
  --line-strong: oklch(100% 0 0 / 0.16);
  --line-focus:  var(--accent);

  /* 强调:见 §6 选定(候选 1 精炼琥珀 / 候选 2 电青)。当墨水,不做辉光背景 */
  --accent:      oklch(80% 0.13 75);   /* 候选1:精炼琥珀(比现在更收敛、更冷一点) */
  --accent-ink:  oklch(18% 0.03 75);   /* 强调底上的字 */
  --accent-quiet: oklch(80% 0.13 75 / 0.12);

  /* 语义色:克制 */
  --positive: oklch(72% 0.13 160);
  --warning:  oklch(80% 0.12 75);
  --danger:   oklch(64% 0.17 25);
  --info:     oklch(70% 0.10 230);
}
```
- 媒体区用 `--bg-sunken` 当衬底,让图/视频"嵌进暗房灯箱"而非浮在卡片上。
- **禁止**:`accent` 当大面积背景、`accent-glow` 模糊辉光、彩虹渐变。强调色单屏出现 ≤3 次。

### 4.2 字体系统(本次重构的最大单点提升)
| 角色 | 拉丁 | 中文 | 用途 |
|---|---|---|---|
| **Display 显示** | Geist / General Sans | **思源黑体 Bold / HarmonyOS Sans SC** | 标题、关键数字 |
| **Text 正文** | Inter / Geist | **思源黑体 Regular / HarmonyOS Sans** | 正文、标签、按钮 |
| **Mono 数据** | Geist Mono / JetBrains Mono | (等宽数字即可) | seed、分辨率、step/cfg、时间码、节点端口 |
| **Serif(仅点缀)** | Fraunces | **思源宋体 Source Han Serif** | 仅登录/营销/封面极少量,**绝不在工作区当主标** |

- **必须给中文配真正的字体**(思源黑/HarmonyOS Sans):这是从"系统苹方放大"到"设计过的中文排版"的质变。通过 `next/font/local` 子集化加载,避免 FOUT + 体积。
- 字阶收敛:display `clamp(1.6rem→2.6rem)`(比现在的 4.4rem 巨标小一档,**让作品而非文字当主角**),text 15px/1.6,mono 13px。
- 抛弃"中文巨型衬线刊头"。标题用 grotesk 中等字号 + 字距 + 层级,而非靠字号轰炸。

### 4.3 间距 · 栅格 · 圆角 · 高度(严格刻度)
- **8pt 基**:`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`,组件内只用这些值。
- **栅格**:工作区 12 列 + 可选 bento;媒体网格用 `auto-fill, minmax(clamp)` 自适应。
- **圆角阶**:`6(控件) / 10(卡片) / 16(面板/模态) / 999(胶囊)`,全站只用这 4 个。
- **高度阶 elevation**(背景 + 光边 + 阴影确定组合):
  - `e0` 页面:无阴影
  - `e1` 卡片:`--bg-raised` + `inset 0 0 0 1px line` + `shadow-sm`
  - `e2` 悬浮卡:`+ shadow-md` + `line-strong`
  - `e3` 弹层/模态:`--bg-overlay` + `shadow-lg` + 背景 `backdrop-blur(12px)`
  - `e4` 拖拽中:`+ shadow-xl` + `scale(1.02)`

### 4.4 动效(语义化 · 仅合成器属性 · 尊重 reduced-motion)
- 只动 `transform / opacity / filter / clip-path`;统一两条缓动:`--ease-out: cubic-bezier(0.16,1,0.3,1)`、`--spring`(framer `springSoft`)。
- 语义:**进入**=淡入+8px 上移错峰;**层级**=缩放+阴影;**反馈**=强调色描边脉冲一次;**连续**=骨架流光。
- **把你之前列的交互打磨内置进组件库**:① 生成时**骨架屏/流光占位**;② 结果网格**拖拽排序**;③ 参数**双滑块范围**;④ 悬停**放大镜预览**;⑤ 作品库**懒加载瀑布流**。

### 4.5 组件语言(原子 → 分子,统一重做)
- **按钮**:primary(实心强调,仅每屏一个主行动)/ ghost(描边)/ subtle(透明)/ danger;统一高度 36/44,焦点环 `2px line-focus`。
- **输入 / 文本域 / 下拉 / 分段控**:统一 e1 底 + 聚焦时 `line-focus` + 轻位移。
- **Slider / RangeSlider**:已有 Slider,补**双滑块范围** + 数值气泡。
- **卡片 / 媒体卡**:媒体卡=作品铺满、信息悬停浮现、角标极小;e1→e2 悬停。
- **模态 / 抽屉 / Toast**:e3 玻璃 + 背景压暗;Toast 右下、强调色细边。
- **导航(顶栏重做)**:从"一排同权按钮"改为**主区(创作/画布/漫剧)+ 资源区(作品/模型)+ 系统(管理/退出)分组**,当前态用墨块滑移(已有 `layoutId`);收窄留白、降低存在感。
- **空态**:删序数水印;改"一句话引导 + 一个主行动 + 极轻图形",安静且有用。
- **进度**:真百分比(已有 SSE)+ 骨架流光,统一一套。

### 4.6 关键屏蓝图
1. **外壳 / 导航**:顶栏分组 + 内容区铺满;暗房中性底。
2. **创作台**:左参数(可折叠)/ 中**大舞台(作品主角,铺满)**/ 右续创作;生成时舞台先上骨架流光,再渐显结果。
3. **画布**:已是 React Flow,套用新 token(中性底 + 精确强调连线)+ 节点 e1/e2 + mono 端口标签。
4. **作品库**:**懒加载瀑布流**铺满,悬停放大镜 + 灯箱;筛选条收窄成胶囊。
5. **漫剧导演台**:左流程轨 + 中分镜墙(媒体主角)+ 右属性;导出步成片做"灯箱播放器"。
6. **模型库**:目录式但去巨标;mono 呈现模型元信息。

## 5. 实施计划(分阶段,绝不破坏现有功能)

> 全部用既有 token 变量名做**别名映射**,逐屏切换,每阶段 `build + tsc + 截图`验真。

- **Phase 0 — 地基(原子层,不动业务)**:换字体系统(中文真字体 + mono)、重写 `:root` token(中性色 + 强调收敛 + elevation/刻度)、重做基础组件(按钮/输入/滑块/卡片/模态/Toast)。**此阶段完成,全站观感已质变。**
- **Phase 1 — 外壳 + 创作台**:顶栏分组导航;创作台大舞台 + 骨架流光。
- **Phase 2 — 作品库 + 画布**:瀑布流懒加载 + 放大镜 + 灯箱;画布套新系统。
- **Phase 3 — 漫剧 + 模型库 + 音频/3D/管理/助手**:逐屏迁移,删 gimmick 空态。
- **Phase 4 — 打磨**:拖拽排序、双滑块、微动效、暗/亮双主题(若选 C 作第二主题)、可访问性(对比度/键盘/reduced-motion)、性能(字体子集、玻璃节制)。

**风险与对策**:中文字体加载(子集 + `next/font/local` + `font-display:swap`);色彩迁移(token 别名,逐屏切不一次性炸);组件回归(每阶段截图 + tsc + build)。
