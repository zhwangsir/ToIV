# ToIV 产品现状诊断 vs 理想 — 差距与整合报告
> 2026-06-25 · 应"功能完整度不够 / 重复未融合 / UI 动效没磨透 / R18 打不开"的反馈而作。

---

## 0. 结论先行

**病根**:前期用并行 agent 快速铺广度,**每个面各做各的**——生成、模型选择、提示词优化、NSFW 档、结果展示,各自实现一套,没有共享内核。于是:功能广但浅、重复未融合、各面手感不一致。

**药方**:进入**收敛阶段**——抽公共内核 → 删/并旧脚手架 → 逐模块补完整 → 统一 UI/动效。**不再加新功能**,把现有做成"一个连贯的产品"。

**立即修**:R18/设置入口(根因:设置埋在飘浮岛 hover 展开区,hover 本身不稳 → 点不到)。

## 1. 跨模块「重复未融合」(收敛优先治这些)

| # | 重复点 | 现状(实测) | 应融合为 |
|---|---|---|---|
| A | **生成内核** | `create/useGenerationFeed`、`canvas/useNodeGeneration`、`manju`、`threed`、`audio` 各写一遍 submit→SSE/track→结果 | 单一 `useGeneration` hook(提交 + 服务端追踪已有 tracker + 结果),各面复用 |
| B | **模型选择** | ~7 处各自 `listModels` + 选择 UI(create×3、canvas×4、manju、models) | 一个 `<ModelPicker>` + 单一 models 数据源(SWR 缓存已就位)|
| C | **提示词优化** | ~10 处各调 `/api/optimize`(OptimizeButton、ShotInspector、canvas TextNode/AudioNode、audio…) | 统一一个 `<OptimizeButton>`/hook(内容感知 optimize)|
| D | **NSFW 档** | ~17 文件散落本地态(create×2、canvas 节点×3、nav、library、models) | R18 单一真相源 `NsfwContext`(已建)→ 各处只读 `enabled`,删散落本地态 |
| E | **旧脚手架/死代码** | `components/video/VideoStudio.tsx` **已死**(无视图引用);`components/generate/`(PromptForm/ResultGallery)**旧脚手架**,仍被 3D/音频依赖 | 删 VideoStudio;3D/音频迁到统一内核后删 generate/ |

> E 的含义:**3D / 音频两个面还跑在旧的 `generate/` 脚手架上**,没跟创作台(`create/`)统一 —— 这就是"为什么各部分手感/完整度不一致"的直接证据。

## 2. R18 / 设置打不开(立即修,根因已定位)

**根因**:设置齿轮埋在灵动岛**悬停展开**的账户区里。实测真实鼠标 hover 灵动岛**不稳定展开**(`isExpanded:false`)→ 账户区/齿轮根本不挂载 → 点不到。之前的补丁("打开后保持展开")只解决"开了又消失",没解决"开不出来"。

**修法**:账户/设置改成**常驻可点**——紧凑灵动岛里放一个常驻账户按钮,点击开 **portal 菜单**(R18 开关 / 主题 / 退出),彻底脱离 hover。这是确定的修复,不需讨论。

## 3. 各模块:现状 vs 理想 vs 缺口

### 🎨 创作台(图/视频/3D/音频 主生成)
- **现状**:简易/专业双层;图/视频/3D/音频四模式;参数滑块、模型/LoRA、NSFW 档、内容感知优化、结果流(拖拽排序/放大镜/骨架流光);R18 服务端分区。
- **缺口**:① 续创作(转视频/转 3D/变体/重绘)链路浅;② img2img / inpaint / ControlNet 没打通(ControlNet 仍占位);③ 参数/工作流预设弱;④ 种子/批次/历史管理缺;⑤ **视频/3D/音频模式深度远不如图像**(且 3D/音频走旧 generate/ 脚手架)。
- **理想**:模式自适应的统一创作台,四模式参数都完整,续创作顺滑,与画布/作品库互通。

### 🕸 画布(节点式工作流)
- **现状**:React Flow 节点(文本/图/视频/音频 + 分镜/三视图/打光/3D 结构节点);拖端口连线;运行全部;工作流命名存取;节点 NSFW;产物归档。
- **缺口**:① 节点参数比创作台浅;② 无"局部重跑/中间结果缓存"(改一个节点要整链重跑);③ 无撤销/重做、复制节点等编辑器基本功;④ 无工作流**模板库**;⑤ 与创作台/漫剧资产互通弱。
- **理想**:画布即"专业模式",共享创作台的生成内核与模型/参数,可存取/分享工作流模板,支持局部重算。

### 🎬 漫剧导演台(分镜→视频→成片)
- **现状**:剧本→分镜(LLM)→逐镜出图→关键帧转视频→自动剪辑(ffmpeg crossfade+字幕+BGM,已端到端验证)+ 每镜 AI 润色。视频已输出真 mp4。
- **缺口**:① **角色一致性(IPAdapter)未做** —— 这是 M2 核心,当前每镜各画各的、人物不一致;② 分镜→视频逐镜手动,缺批量/一键流水线;③ 配音/TTS、字幕样式、BGM 库缺;④ 转场/运镜模板缺;⑤ 多漫剧项目管理缺。
- **理想**:输入剧情 → 一键产出**角色一致**的成片(分镜+配音+字幕+转场),可逐镜精修。

### 🖼 作品库 + 🛒 模型市场
- **现状**:作品库瀑布画廊 + 灯箱 + 懒加载 + 类型筛选 + R18 分区;市场 Civitai/HF 搜索(走 civitai.red + key)。
- **缺口**:① 作品无搜索/标签/收藏/分组/项目归属;② 作品→再创作(作参考图/复用 seed)链路缺;③ **市场只搜不落地**(下载模型到 worker 集群没打通);④ 无"一键装模型到集群"。
- **理想**:作品库=创作资产中枢(搜/管/再用);市场能一键把模型装到 ComfyUI 集群。

## 4. UI / 动效磨合

- 灵动岛性能刚优化(memo + layout 收敛,只活动行随进度重渲);但**视图间是硬切**(无过场转场);加载骨架/空态/hover 态across 各面仍不完全统一。
- 动效语言要统一到一套(`springSoft`);窄屏/移动端未系统验证;a11y(键盘全链路、对比度)需终检。

---

## 5. 建议执行序(收敛阶段)

1. **立即**:修 R18/设置入口(常驻点击 portal 菜单)。
2. **清场**:删 VideoStudio 死代码;标记 generate/ 待并。
3. **抽内核**:`useGeneration` 生成 hook · `<ModelPicker>` · `<OptimizeButton>` · NSFW 单源 · 统一结果视图 → 各面接入(顺带把 3D/音频从旧 generate/ 迁过来)。
4. **补完整**(按你定优先级,均在列):创作台续创作+ControlNet → 画布局部重跑+模板 → **漫剧 IPAdapter 角色一致性** → 库搜索/再用 + 市场下载落地。
5. **磨 UI/动效**:视图过场 + 统一空态/骨架/hover + 窄屏 + a11y 终检。

> 整合(3)是"融合重复"的关键,且让后续每模块补完整事半功倍(各面共享内核)。漫剧 IPAdapter、市场下载落地 是"补完整"里最实的两块。
